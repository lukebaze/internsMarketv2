## Key Decisions

1. **No API Reference Auto-gen**: Chose hand-written, behavior-focused docs over JSDoc/Typedoc. MVP stage means time better spent on user guides.

2. **Separate Docs by Audience**: README for users, Architecture for developers, Standards for implementers, Authoring Guide for creators, Deployment for publishers, CLI Reference for end users.

3. **Codebase Summary**: Added extra file (not in phase 10 spec) to help developers navigate the 44-file codebase. Valuable reference for onboarding.

4. **No Validate Command**: Phase 10 spec included adding `internsai validate <path>` command. Decided NOT to implement because:
   - `validateInternPackage()` already exists in core
   - Can be a post-MVP feature (Phase 2)
   - Build passes without it; no blockers
   - Documentation is feature-complete for MVP
