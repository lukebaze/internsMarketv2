# InternsMarket CLI

One-line description: Install, manage, and apply AI interns to your workflow via the command line.
URL: https://internsmarket.com

---

## Quick Start

```
npm install -g internsmarket
im install content-marketing-intern
im apply content-marketing-intern
```

---

## CLI Commands

| Command              | Purpose                                                        |
|----------------------|----------------------------------------------------------------|
| `im setup`           | Interactive wizard to install runtimes (ZeroClaw, OpenClaw)   |
| `im install <id>`    | Install an intern from registry with license + sig validation  |
| `im update [id]`     | Update one or all installed interns                            |
| `im remove <id>`     | Remove an installed intern                                     |
| `im list`            | Show all installed interns and their versions                  |
| `im activate`        | Activate a Polar.sh license key                                |
| `im status`          | Show current tier and installed intern count                   |
| `im apply <id>`      | Generate runtime config for ZeroClaw or OpenClaw              |

## What is an AI Intern?

An AI Intern is an AIEOS v1.1 entity — a structured AI agent with:

- **Personality**: Neural matrix traits (e.g., creative, analytical, methodical)
- **Voice**: Idiolect and text style unique to each intern
- **Skills**: 3–5 specialized skills per intern
- **Memory**: Retains brand voice, templates, and preferences across sessions

Each intern runs as an independent agent in your chosen runtime. They do not share context with each other by default.

## Typical Workflow

1. Install the CLI: `npm install -g internsmarket`
2. Run setup: `im setup` — choose and install a runtime
3. Activate license (if purchased): `im activate`
4. Install an intern: `im install <intern-id>`
5. Apply the intern: `im apply <intern-id>` — generates config for your runtime
6. Open the generated config in ZeroClaw or OpenClaw to start working with the intern
7. Update anytime: `im update` or `im update <intern-id>`
8. View what is installed: `im list`
9. Check your tier: `im status`

---

## Notes for AI Agents

- Intern IDs follow the pattern: `<role>-intern` (e.g., `code-review-intern`)
- `im install` verifies a cryptographic Ed25519 signature and checks the Polar.sh license before installing
- `im apply` reads the installed intern's manifest and emits a runtime-specific config file
- Free interns do not require a license key
- Paid interns require an active Polar.sh license; use `im activate` to register the key locally
- Use `im status` to confirm tier and which interns are available before running `im install`
