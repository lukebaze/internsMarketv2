## Language Rules

**Person:**
[ ] Second person ("you log in") — recommended
[ ] Third person ("the user logs in")

**Active vs. passive:**
[ ] Active voice preferred ("Click Save" not "Save should be clicked")

**Words we use:**
_e.g. "workspace" not "project", "sign in" not "log in"_

| Preferred | Avoid |
|-----------|-------|
| | |

**Forbidden phrases:**
_e.g. "simply", "just", "easy", "obviously"_

## Structure Rules

**Page structure:**
- Every page starts with: [ ] What you'll learn [ ] Overview [ ] Prerequisites

**Step format:**
- One action per step
- Steps use imperative: "Click", "Enter", "Select" — not "You should click"

**Code examples:**
- [ ] Always include working examples (no pseudocode)
- Language: specify if always one language, or note that examples match the reader's stack

**Screenshots:**
- [ ] Include for UI flows [ ] Text only [ ] Both
